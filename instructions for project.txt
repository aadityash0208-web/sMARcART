### 2. Mongoose Schemas (Strictly Following Page 10 Data Dictionaries)

#### User Model (`server/models/User.js`)
- **Attributes**: UserID (auto-generated _id), Name, Email, Password, Address
- **Description**: Stores customer details
- **Features**:
  - Email validation and uniqueness
  - Password minimum length validation
  - Timestamps (createdAt, updatedAt)

#### Product Model (`server/models/Product.js`)
- **Attributes**: ProductID (auto-generated _id), Name, Description, Price, Stock, CategoryID
- **Description**: Stores product information
- **Features**:
  - Price and Stock validation (non-negative)
  - Reference to Category model
  - Timestamps

#### Category Model (`server/models/Category.js`)
- **Attributes**: CategoryID (auto-generated _id), CategoryName
- **Description**: Product classification
- **Features**:
  - Unique category names
  - Timestamps

#### Order Model (`server/models/Order.js`)
- **Attributes**: OrderID (auto-generated _id), UserID, ProductID, Quantity, Status, OrderDate
- **Description**: Stores order details
- **Features**:
  - References to User and Product models
  - Status enum: pending, shipped, delivered, cancelled
  - Automatic OrderDate with Date.now default
  - Timestamps

## How to Use This Project

### For Your University Project

1. **Copy the Project**:
   - Download all files from `/app/client` and `/app/server` directories
   - Transfer to your local machine

2. **Backend Setup**:
   ```bash
   cd server
   npm install
   cp .env.example .env
   # Edit .env with your MongoDB connection string
   npm run dev
   ```

3. **Frontend Setup**:
   ```bash
   cd client
   npm install
   npm start
   ```

4. **Database Setup**:
   - Install MongoDB locally or use MongoDB Atlas
   - Update MONGO_URI in server/.env
   - Models will auto-create collections on first use
